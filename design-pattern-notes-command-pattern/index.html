什么是命令模式

在一些如使用请求队列的场景中，请求的执行者和请求的发出者不是一个，而具体的请求（行为）差异很大.

命令模式，将具体的请求（这里的请求可以认为是一些行为），抽象为对象，来达到发出请求的对象和执行请求的对象之间的解耦。

UML 图以及说明

  • Receiver：具体的请求对象，action 方法是请求执行的具体行为。当然，其他请求的行为不一定是 action 方法。
  • Command：是抽象的命令接口。
  • ConcreteCommand：某个具体的 Command，里面包装了一个 Receiver。
  • Client：用来将 Receiver 封装为 Command。
  • Invoker：请求的执行者。在这里 Invoker 不必关注 Receiver 的具体细节，只执行 Command 的 execute 方法即可，与 Receiver 和 Client 解耦。

代码示例

 1package designpattern
 2
 3import "fmt"
 4
 5//Command 命令的抽象接口
 6//不考虑撤销等操作的话，也可以直接用闭包
 7type Command interface {
 8	Exe()
 9}
10
11//Invoker 执行 Command
12type Invoker struct {
13}
14
15//Run 执行 Command 的 Exe
16func (i Invoker) Run(c Command) {
17	c.Exe()
18}
19
20//ReceiverA 具体的 Receiver 对象
21type ReceiverA struct {
22}
23
24//Action 具体的行为
25func (r ReceiverA) Action() {
26	fmt.Println("receiver a action")
27}
28
29//ReceiverB 具体的 Receiver 对象
30type ReceiverB struct {
31}
32
33//Action1 具体行为
34func (r ReceiverB) Action1() {
35	fmt.Println("receiver b action 1")
36}
37
38//Action2 具体行为
39func (r ReceiverB) Action2() {
40	fmt.Println("receiver b action 2")
41}
42
43//CommandA 具体的 Command
44type CommandA struct {
45	r ReceiverA
46}
47
48//Exe 执行具体的请求
49func (c CommandA) Exe() {
50	c.r.Action()
51}
52
53//CommandB 具体的 Command
54type CommandB struct {
55	r ReceiverB
56}
57
58//Exe 执行具体请求
59func (c CommandB) Exe() {
60	c.r.Action1()
61	c.r.Action2()
62}
63
64//NewCommandA 生成 CommandA 实例
65func NewCommandA(r ReceiverA) Command {
66	return CommandA{
67		r: r,
68	}
69}
70
71//NewCommandB 生成 CommandA 实例
72func NewCommandB(r ReceiverB) Command {
73	return CommandB{
74		r: r,
75	}
76}
77
